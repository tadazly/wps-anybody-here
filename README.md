# 表里有人

> 看看表里谁在配，谁在改，谁在和你撞格子

WPS 表格协作提醒插件。它不替代 WPS 云协作，只解决团队在同一个 Git 仓库里编辑本地配置表时“我不知道别人也在改”的问题。

第一版能力：

- 显示同一张表格房间里的在线用户。
- 有人加入或离开时，在任务窗格顶部弹提示。
- 同步其他人当前选中的单元格。
- 用固定用户颜色高亮远端选区。
- 不同用户修改同一个 `sheetName + address` 时，把该单元格标为冲突。
- 服务端关闭后，插件显示“发布机已下线”，并每 10 秒自动重连。

## 目录

- `packages/addin/`：WPS 表格加载项。
- `packages/server/`：中心 WebSocket 服务，建议跑在团队都能访问的发布机或内网机器上。
- `packages/shared/`：插件和服务端共享的协议类型、用户颜色等工具。
- `docs/implementation.md`：最初的大纲和后续升级方向。
- `docs/DECISIONS.md`：关键设计决策，后续改动前先看这里。

## 给发布机同事

在发布机上启动服务：

```bash
npm install
npm run build
npm run dev:server
```

默认监听：

```txt
ws://127.0.0.1:18080
http://127.0.0.1:18080/health
```

发布机看板：

```txt
http://127.0.0.1:18080/
```

这个网页可以看到当前在线用户、打开的表格、每张表是谁在用，以及本次服务启动后的编辑贡献度。

如果同事要从其他电脑连接，把插件里的服务器地址改成发布机内网 IP，例如：

```txt
ws://192.168.1.20:18080
```

## 给策划同事

1. 打开 WPS 表格。
2. 打开 `表里有人` 选项卡。
3. 点击 `协作看板`。
4. 第一次使用时填写服务器地址、表格仓库地址、本地仓库根目录和自己的名字。
5. 保存后会自动加入协作。

加入后，你会看到：

- 在线用户：当前还有谁打开了同一张表。
- 远端选区：别人正在看或编辑哪里。
- 冲突：如果你和别人改了同一个单元格，这里会出现红色记录。
- 标记导航：点击彩色标记可跳到对应单元格。

## 当前限制

- 第一版按单元格地址判断冲突，例如 `Sheet1!B20`。如果表格排序或插行，后续要升级为 `id + field` 级冲突判断。
- 第一版高亮通过设置单元格底色实现，会尽量恢复原底色，但复杂样式仍建议后续改成边框或浮层。
- WPS 的事件绑定和不同版本有关，已经按类型声明接入 `SheetSelectionChange` / `SheetChange`，仍需要在真实 WPS 环境里做最终确认。

## 开发验证

服务端：

```bash
npm run build --workspace @wps-anybody-here/server
```

插件：

```bash
npm run build --workspace @wps-anybody-here/addin
```

开发时可以先启动服务端，再启动插件 dev server：

```bash
npm run dev:server
npm run dev:addin
```
